항상 헷갈리는, 아니 제대로 알았던 적도 없던 개념들

애네가 이제 클래스 쪽에서 나오는 개념인데, 클래스란건 다시 설명하자면 데이터와 함수를 묶어 놓은 object이다. 이러한 클래스 object는 2가지 연산을 한다.

일단 첫번째로, 얘네는 attribute 참조를 하고

두번째로, 얘네는 instance object를 생성할 수 있다.

우리가 예를 들어, numpy.array라는걸 만든다면, 이게 꼴이 "class이름.attribute이름"인거다. 클래스 안의 모든 함수와 변수는 attribute다. 심지어 클래스 안의 __init__이나 __call__등도 함수이므로 얘네도 attribute다.

즉 우리가 method라고 부르는 object도, class object안에 포함된 함수이므로 attribute가 method를 포괄하는 개념이다.

인스턴스란 놈은, 클래스로 만든 object이다. 다르게 말하면 class의 클론이다. 이 인스턴스가 할 수 있는 유일한 연산은 attribute 참조뿐이다.
